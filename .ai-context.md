# AI Agents & Skills Architecture (CRITICAL)

To prevent duplication and ensure consistency across multiple AI tools (Cursor, Windsurf, Roo, Claude, etc.), this project uses a CENTRALIZED architecture for skills and agents.

## 1. Centralized Storage
- **Skills:** All skills MUST be stored in the `/.shared-skills/` directory.
- **Agents:** All agent definitions MUST be stored in the `/.shared-agents/` directory.

## 2. Directory Junctions (Links)
The local `skills/` and `agents/` folders inside tool-specific directories (e.g., `.cursor/skills`, `.agent/agents`, `.windsurf/skills`, etc.) are **Windows Junctions** pointing to the central shared folders.

## 3. Mandatory Rules for AI Agents
When asked to ADD or UPDATE a skill or agent:
- **ALWAYS** perform the operation in the central folders: `/.shared-skills/` or `/.shared-agents/`.
- **NEVER** delete a tool-specific folder (like `.cursor/skills`) to replace it with a regular folder. If the Junction is missing, recreate it using `cmd /c mklink /j <local-path> <shared-path>`.
- **DO NOT** create duplicates. If a skill or agent exists in the shared folder, update it there.
- All tools see the same files. A change in one is a change in all.
